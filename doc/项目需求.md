# 客服管理系统需求
## 项目背景：
公司主要业务是对公共卫生行业提供软件服务，本项目主要解决软件客户在使用软件的过程中，遇到的问题，并提供相应的解决方案。
由于客户端电脑网络受限，一般情况下的沟通是个人通过自己的手机联系到客服进行沟通问题，如果需要远程协助，需要通过VPN连接到具体的电脑进行协助。
## 项目目标：
1. 客户能够通过远程沟通，通过在线咨询方式解决问题。
2. 系统需要具备一般IM功能，包括发送消息、接收消息、在线状态、消息记录等。
3. 系统需要具备客服管理功能，包括客服管理、客户管理、会话管理、工单管理等。
4. 系统需要具备数据分析功能，包括客户分析、会话分析、工单分析等。
## 项目需求：
1. 用户注册与登录
2. 角色与权限管理
3. 组织架构管理
- 支持多层级部门结构
- 客服人员分组管理
- 支持跨部门协作
4. 客户信息管理
- 客户基本信息维护(姓名、电话、邮箱、地址等)，并可以和OA中的客户档案信息对接
- 客户标签分类(普通客户、VIP客户)
- 客户历史服务记录查询
5. 会话管理
- 支持在线聊天窗口
- 多会话并发处理
- 会话转接和转接记录
- 会话排队管理
- 快捷回复功能
- 消息已读/未读状态
6. 工单管理
- 工单创建
- 工单跟进
- 工单分配
- 工单跟进记录
- 工单状态管理
7. 系统集成
- 能够提供接口，方便公司的系统集成，客户可以通过系统集成接口，直接在自己的系统中使用客服系统
8. 知识库管理
- 知识库文章创建与维护
- 知识库分类管理
- 知识库搜索功能
9. 智能辅助
- 智能问答辅助，帮助客服人员快速查找答案，生成话术
10. 数据分析
- 客户分析
- 会话分析
- 工单分析
